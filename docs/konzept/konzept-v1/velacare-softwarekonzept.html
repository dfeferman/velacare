Kapitel 1
Übersicht & Ziele

Was gebaut wird

Eine Web-App in zwei Bereichen

Velacare benötigt eine vollständige Web-Applikation mit zwei klar getrennten Bereichen: ein Kundenportal für Pflegebedürftige und Angehörige, und ein Admin-Panel für den Betrieb. Version 1 funktioniert ohne direkte Kassenkommunikation — alle Anfragen werden manuell durch das Velacare-Team bearbeitet.

Scope Version 1
Registrierung, Anmeldung, Box zusammenstellen, Bestellungen verwalten, Kundenverwaltung im Admin, Produktverwaltung. Keine automatisierte Pflegekassen-Kommunikation.
Für Version 2 vorgemerkt
Automatische Antragstellung bei Pflegekassen, digitale Unterschrift, Kassenschnittstellen, Rechnungsstellung, Partnerportal für Pflegedienste.
🌐

Öffentliche Website

Marketing-Seiten (Startseite, Wie es funktioniert, Produkte, FAQ, Über uns, Kontakt). Kein Login erforderlich. Führt zum Registrierungsformular.

👤

Kundenportal

Nach Registrierung/Login. Box verwalten, Lieferung steuern, Kontaktdaten ändern, Anfragen stellen, Account löschen.

⚙️

Admin-Panel

Nur für Velacare-Mitarbeiter. Kundenverwaltung, Produktverwaltung, Bestellübersicht, Anfragen bearbeiten, Lieferungen steuern.

Kapitel 3
Kundenportal

Registrierung & Onboarding

4-Schritt Antragsprozess

Orientiert an frontida.de und pflegetipp.de: Der Nutzer durchläuft einen geführten Funnel. Am Ende landet die Anfrage als neuer Kundensatz im Admin — kein automatischer Versand, keine Kassenkommunikation in V1.

Registrierungs-Flow (Öffentlich → Konto)4 Schritte
1
Anspruch prüfen
Pflegegrad (1–5) wählen · Pflegeort bestätigen (zuhause) · Versicherungsart (gesetzlich/privat). System zeigt sofort: „Sie haben Anspruch auf bis zu 42 €/Monat." Bei fehlendem Anspruch: freundliche Erklärung + Weiterleitung.
2
Persönliche Daten erfassen
Pflegebedürftiger: Vorname, Nachname, Geburtsdatum, Adresse, PLZ/Ort · Antragsteller (optional, wenn Angehöriger): Vorname, Nachname, Verhältnis · Krankenkasse (Freitext oder Dropdown mit häufigen Kassen) · Telefonnummer · E-Mail-Adresse · Passwort (für Konto-Erstellung).
3
Box zusammenstellen
Interaktiver Konfigurator (siehe Kapitel 4) · Budget-Counter live (0–42€) · Produkte wählen per Checkbox/Toggle · Box kann jederzeit später im Konto geändert werden.
4
Bestätigung & Konto-Aktivierung
Zusammenfassung der Angaben · DSGVO-Einwilligung + AGB (Pflichtfelder) · Wunsch-Lieferstichtag wählen (1.–28. eines Monats) · Absenden → Bestätigungs-E-Mail mit Aktivierungslink · Admin wird benachrichtigt (neue Kundenanfrage).

Kapitel 7
Techstack & Architektur

Technologie-Entscheidungen

Modern, wartbar, skalierbar

Der Stack ist bewusst pragmatisch gewählt — keine Over-Engineering, aber zukunftssicher genug für V2 (Kassenkommunikation, höhere Last). Alle Entscheidungen sind begründet.

Frontend
Next.js 14 (React)
Server-Side Rendering für SEO der öffentlichen Seiten, App Router für Kundenportal. TypeScript für Typsicherheit. Gleicher Stack für Website + Portal.
Styling
Tailwind CSS
Schnelle Umsetzung des Velacare Design-Systems via CSS-Variablen. Kein separates CSS-Framework nötig. Cormorant + DM Sans via Google Fonts.
Backend
Next.js API Routes
Für V1 reichen API-Routes im gleichen Next.js-Projekt. Bei V2 (Kassenschnittstellen) einfache Migration zu eigenem Node.js/Express-Dienst möglich.
Datenbank
PostgreSQL
Relationale Datenbank mit JSON-Support (JSONB für Box-Snapshots). Robust, DSGVO-konform, gut unterstützt. ORM: Prisma für typsichere Queries.
Auth
NextAuth.js
E-Mail/Passwort-Login für V1. Session-Management, JWT, einfache Rollenunterscheidung (kunde/admin/superadmin). Für V2 erweiterbar um OAuth (Google/Apple).
Datei-Upload
Cloudinary / S3
Produktfotos extern gespeichert, automatische Komprimierung, CDN-Auslieferung. Cloudinary für einfachen Start, S3 wenn DSGVO-konformes deutsches Hosting nötig.
E-Mail
Resend / SendGrid
Transaktionale E-Mails (Bestätigung, Lieferstatus, Anfragen-Antworten). Templates in React-Email oder HTML. DSGVO: Anbieter mit EU-Servern wählen.
Hosting
Vercel + Supabase
Vercel für Next.js (Zero-Config-Deployment, automatische Preview-Deployments). Supabase als managed PostgreSQL mit EU-Region (Frankfurt). Günstig für V1, skalierbar.
Monitoring
Sentry
Error-Tracking Frontend + Backend. Kostenlos bis 5.000 Errors/Monat — ausreichend für V1.
Versionierung
Git + GitHub
Branches: main (Production), develop (Staging), feature/* (Features). GitHub Actions für CI/CD-Pipeline mit automatischen Tests vor Deployment.

Kapitel 8
Sicherheit & DSGVO

Pflichtanforderungen

Sensible Daten — hoher Schutz

Da Velacare Gesundheitsdaten (Pflegegrad) und personenbezogene Daten älterer Menschen verarbeitet, gelten erhöhte Sicherheitsanforderungen gemäß DSGVO.

Authentifizierung

bcrypt Passwort-Hashing

Passwörter werden niemals im Klartext gespeichert. bcrypt mit Salt-Rounds ≥ 12.

E-Mail-Verifikation

Konto ist erst nach Klick auf Bestätigungslink vollständig aktiv. Token läuft nach 24h ab.

Rate Limiting Login

Max. 5 Fehlversuche, danach 15min Sperrung. Schutz vor Brute-Force.

Session-Timeout

Automatischer Logout nach 30min Inaktivität (Kundenkonto), 60min (Admin).

Passwort-Reset

Token per E-Mail, einmalig gültig, läuft nach 1h ab.

Datenschutz (DSGVO)

Datenminimierung

Nur notwendige Felder werden erhoben. Kein Tracking ohne Einwilligung.

Recht auf Löschung

Kunde kann Account selbst löschen → Soft-Delete, Anonymisierung nach 30 Tagen, Lieferprotokolle bleiben anonymisiert erhalten.

Recht auf Auskunft

Admin kann auf Anfrage alle gespeicherten Daten eines Kunden als PDF/JSON exportieren.

Einwilligung dokumentiert

DSGVO-Einwilligung + AGB-Zustimmung werden mit Timestamp gespeichert.

Hosting in der EU

Datenbank in Frankfurt (Supabase EU-Region). Kein Datentransfer in Drittländer.

Zugriffsschutz

Route-Guards (Middleware)

/konto/* nur für eingeloggte Kunden. /admin/* nur für Admin/Superadmin. Redirect auf Login bei fehlendem Token.

Kunden sehen nur ihre Daten

API-Endpoints prüfen immer ob die angefragte Ressource dem eingeloggten User gehört.

HTTPS erzwungen

HTTP-Anfragen werden auf HTTPS umgeleitet. HSTS-Header gesetzt.

CSRF-Schutz

NextAuth setzt CSRF-Token bei allen Formularen automatisch.

Daten-Integrität

Audit-Log unveränderlich

INSERT-only Tabelle. Kein UPDATE/DELETE über API möglich. Nur via direktem DB-Zugriff (nur Superadmin).

Box-Snapshots bei Lieferung

Bei jeder Lieferung wird die Box-Konfiguration eingefroren (JSONB). Spätere Änderungen beeinflussen vergangene Lieferungen nicht.

Input-Validierung

Alle Formulareingaben serverseitig validiert (Zod). Kein blindes Vertrauen auf Client-Daten.

Datenbank-Backups

Tägliche automatische Backups (Supabase), 30 Tage Aufbewahrung.
